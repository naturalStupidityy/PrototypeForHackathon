Executive Summary

India's Ayush sector stands at a pivotal moment in its digital transformation journey. As the Ministry of Ayush drives the transition from paper-based records to interoperable digital health systems, the integration of traditional medicine terminologies with global standards becomes paramount.

Core Innovation: Dual-Coding Framework

This implementation guide presents a lightweight, FHIR R4-compliant terminology micro-service that bridges the gap between India's traditional medicine systems and global healthcare standards. By integrating NAMASTE codes (4,500+ terms) with WHO's ICD-11 TM2 framework (529 disorder categories), we enable healthcare practitioners to preserve clinical richness while ensuring interoperability.

Strategic Alignment

This solution directly supports the Ayushman Bharat Digital Mission (ABDM) and aligns with India's 2016 EHR Standards, mandating FHIR R4 APIs, SNOMED CT semantics, and ABHA-linked authentication1. The micro-service architecture ensures seamless integration with existing EMR systems while maintaining compliance with national health stack requirements.

Core FHIR R4 Terminology Resources

The terminology micro-service is built around three fundamental FHIR R4 resources that provide standardized representation and management of NAMASTE and ICD-11 terminologies. These resources ensure interoperability while maintaining the clinical richness of traditional medicine systems.

CodeSystem

Defines the set of codes and their meanings for NAMASTE terminology, enabling validation and auto-complete functionality.

URL: http://namaste.ayush.gov.in/fhir/CodeSystem/namaste-disorders

ConceptMap

Defines relationships between NAMASTE codes and ICD-11 codes, enabling translation and dual-coding functionality.

Mappings:
NAMASTE → ICD-11 TM2 NAMASTE → ICD-11 Biomedicine

ValueSet

Groups codes from multiple CodeSystems for specific use cases like auto-complete functionality and search filtering.

Purpose:
• Auto-complete search • Code validation • Filtering and grouping

API Endpoints & Operations

The terminology micro-service exposes a comprehensive set of RESTful API endpoints implementing standard FHIR terminology operations. These endpoints provide the core functionality for code lookup, translation, validation, and secure data exchange.

Auto-Complete Value-Set Lookup

Endpoint

GET /ValueSet/$expand

Key Parameters

url ValueSet canonical URL
filter Search text for auto-complete
count Number of results to return
offset Pagination offset

Example Request

GET /ValueSet/$expand
  ?url=http://namaste.ayush.gov.in/fhir/ValueSet/diagnosis-autocomplete
  &filter=arth
  &count=10

Response Structure

{
  "resourceType": "ValueSet",
  "expansion": {
    "contains": [
      {
        "system": "http://namaste.ayush.gov.in/fhir/CodeSystem/namaste-disorders",
        "code": "AY001",
        "display": "Ama Vata (Rheumatoid Arthritis)"
      },
      {
        "system": "http://id.who.int/icd/release/11/tm2",
        "code": "TM2001",
        "display": "Arthritis, specified"
      }
    ]
  }
}

Translation Operation (NAMASTE ↔ TM2)

Endpoint

GET /ConceptMap/$translate

Key Parameters

url ConceptMap canonical URL
code Source code to translate
system Source code system
targetSystem Target code system

Example Request

GET /ConceptMap/$translate
  ?url=http://namaste.ayush.gov.in/fhir/ConceptMap/namaste-to-icd11-tm2
  &code=AY001
  &system=http://namaste.ayush.gov.in/fhir/CodeSystem/namaste-disorders
  &targetSystem=http://id.who.int/icd/release/11/tm2

Response Structure

{
  "resourceType": "Parameters",
  "parameter": [
    {
      "name": "result",
      "valueBoolean": true
    },
    {
      "name": "match",
      "part": [
        {
          "name": "concept",
          "valueCoding": {
            "system": "http://id.who.int/icd/release/11/tm2",
            "code": "TM2001",
            "display": "Arthritis, specified"
          }
        },
        {
          "name": "equivalence",
          "valueCode": "equivalent"
        }
      ]
    }
  ]
}
